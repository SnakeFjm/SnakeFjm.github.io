网络及AFNetWork框架

  • AFNetWork
    1. 不支持断点下载
    2. 文件上传支持
    3. 扩散

  • 结构
    1. AFURLRequestSerialization 负责 请求request（参数配置， 请求头 和 请求体） 即处理请求的数据配置
    2. AFURLResponseSerialization 响应response（响应参数， 响应头 和 响应体）,即处理网络返回的数据(这个文件，针对不同的业务有不同的子类来处理)
    3. AFURLSessionManager 负责管理 URLSession操作和代理
    4. AFHTTPSessionManager继承AFURLSessionManager， 对外提供使用的类, 并包含了 整个框架的业务属性（集 1，2，3一起）
  • HTTPS
    HTTPS = HTTP + 加密 + 认证 + 完整性保护，其实HTTPS是身披SSL外壳的HTTP
    常用加密方法：
    1. 共享密钥加密（也称为对称加密）：共享密钥加密就是加密和解密通用一个密钥。
      1. 优点是加密解密速度快，缺点是一旦密钥泄露，别人也能解密数据。
    2. 公开密钥加密（也称非对称加密 ），恰恰能解决共享密钥加密的困难， 其过程是这样的：
      加密用公钥，解密用私钥
      1. 发文方使用对方的公开密钥进行加密
      2. 接受方在使用自己的私有密钥进行解密
        就算知道了结果和公钥，破解出被机密的数据是非常难的
        通过公钥加密的内容一定要使用私钥才能够解密。
