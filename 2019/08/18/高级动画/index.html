  • position和anchorPoint
    1. anchorPoint是锚点,它的值范围为(0~1)，是相对layer本身点坐标。
    2. view的旋转会围绕着anchorPoint来旋转
    3. position是layer层相对父层点坐标,position和anchorPoint实际指的是同一个点
    4. position和anchorPoint的改变,对另一个值没影响
    5. anchorPoint值的修改,可以改变view的布局

总结:知道position的值,把锚点这个点和position对齐,就是view的布局

  • 推导frame的计算公式(不包含旋转角度)
    • frame.origin.x = position.x - anchorPoint.x * width
    • frame.origin.y = position.y - anchorPoint.y * height
    • frame.width
    • frame.height

  • 动画的流程
    1. 有delegate，执行delegate
    2. 没有的话寻找actions里的action
    3. 然后再寻找style里的action:defaultActionForKey
    4. 调用actionForKey
    5. 如果有action返回，那么执行addAnimationForKey方法
      上述流程，如果返回为nil，继续走流程，为NSNull null就停止
  • layer层三层树结构
    • presentationLayer Tree（呈现树） 用来展示的
    • modelLayer Tree（模型树） 来存储数据和值的
    • Render Tree（渲染树）

动画的时候是对presentLayer操作，modelLayer还是保持原来的值

模型树：是用来存储数据的、响应事件
呈现树：用来根据模型树的数据，展示
呈现树是对模型树的copy

如果说：你添加了一个动画，那么此时呈现树的数据来自于动画，而不是模型树；当动画结束之后，才又会根据模型树的数据展示；如果动画结束了，但是动画不移除，那么呈现树的数据依然来自于动画

引申：每一个runloop循环的时候，会提交一个事务：CATransaction，让呈现树根据数据展示

  • 动画代理
    代理里面的animOne是animation的深拷贝，不是同一个地址的动画对象
  • 贝塞尔曲线
    • shapelayer.fillRule:
      1. kCAFillRuleEvenOdd 字面意思是“奇偶”。按该规则，要判断一个点是否在图形内，从该点作任意方向的一条射线，然后检测射线与图形路径的交点的数量。如果结果是奇数则认为点在内部，是偶数则认为点在外部
      2. kCAFillRulenonzero 字面意思是“非零”。按该规则，要判断一个点是否在图形内，从该点作任意方向的一条射线，然后检测射线与图形路径的交点情况。从0开始计数，路径从左向右穿过射线则计数加1，从右向左穿过射线则计数减1。得出计数结果后，如果结果是0，则认为点在图形外部，否则认为在内部
